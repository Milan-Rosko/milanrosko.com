Summary

A learning schema in which training is constituted by the elimination of explicit refutation witnesses against a finitely declared semantic object. Each update is triggered by an exhibited margin failure; the process terminates when no such witness remains in the declared obligation set. Convergence is therefore not asymptotic but certificated: closure holds when a decidable predicate over the finite table returns no counterexample.

Glossary

  • Semantics lock

    The declared and fixed preprocessing pipeline, feature schema, scoring rule, readout function, and scan schedule. These are the parameters conventionally left implicit; here they are made explicit and held invariant across the comparison.

  • Witness table

    The enumerated collection of examples over which all margin obligations are checked. The table is the entirety of the declared semantic domain.

  • Refutation witness

    A concrete example/class pair demonstrating that the current weight state violates the declared margin condition. Its existence constitutes a refutation of the hypothesis that learning has terminated.

  • Repair

    The integer weight update triggered by a refutation witness. The repair magnitude is derived from the witness's slack value and is sufficient to eliminate that witness from the obligation set.

  • Closure

    The state in which the declared predicate FULL.ok holds—no refutation witness remains in the finite table. Closure is a certificate, not an approximation.

  • Trace

    The complete, replayable sequence of states produced by successive repairs. Each transition in the trace is justified by an explicit witness and is independently checkable.

On Refutational Learning

The central thesis is that learning can be given a constructive, proof-theoretic account: training proceeds not by descending a scalar objective but by eliminating explicit counterexamples to a declared semantic obligation. When the current interpretation fails a margin condition, that failure is exhibited by a refutation witness—a concrete example/class pair identifying the precise locus of violation.

The learner responds with a local integer repair whose magnitude is derived analytically from the witness. Iterating this procedure produces a finite, replayable trace in which every state transition is grounded in an explicit logical event. Termination coincides with closure: when no refutation witness remains, the interpretation satisfies all declared margin obligations on the finite table, and this fact is certified by a decidable predicate rather than inferred from a loss curve.

The comparison with Gradient Descent is therefore analytic rather than competitive. It concerns two distinct conceptions of what a training step is: the elimination of an explicit semantic failure on one hand, and numerical descent over an aggregate scalar objective on the other.

This is not a benchmark claim. No assertion is made about efficiency or generalisation performance. The comparison isolates what each training trace makes visible to inspection and what kind of evidence it preserves.

SGD is included because it compresses failure into an undifferentiated numerical signal. Refutational Learning retains the failing obligation as a named, inspectable object—a witness that can be stored, replayed, and independently verified.

Note. The interactive demo is most legible after the formal setup below has been read.

Both methods are implemented in JavaScript and evaluated under the same locked prediction rule $\arg\max\nolimits^{*}$. The interface is designed to be analytic rather than competitive. The shared readout isolates the difference between repairing a finite interpretation in response to a named refutation witness and updating real-valued coordinates according to a differentiable objective.

The extensional interface is held fixed throughout, and each trace executes under its declared scan schedule. The salient distinction is not solely how the update is computed, but what evidential residue the update leaves behind: a named witness and an integer delta, versus a gradient and a floating-point step.

(i) The witness table, feature schema (bias and unary indicator facts), class count, and scan order are fixed before either trace is generated. Each cycle therefore has an unambiguous definition and can be replayed or independently re-executed from the stored state.

(ii) Refutational Learning locates the earliest refutation witness to a margin violation, performs a local integer repair on the sparse weight table $W$, and propagates its semantic consequences through precomputed postings into the cached score matrix $S$. Gradient Descent performs SGD updates on the softmax cross-entropy loss with real-valued weights; prediction is still read out by the same locked $\arg\max\nolimits^{*}$, preserving comparability.

What the trace records. The Refutational Learning trace makes explicit each refutation witness, the repair it induces, and the resulting state together with its decidable predicates. The GD trace reports the aggregate loss and accuracy computed under the same locked readout at each cycle.

We begin with linearly separable Boolean functions,

$$\mathrm{NAND}, \mathrm{OR}, \mathrm{AND}$$

for which finite-table closure is attainable and the repair sequence converges to a certificate. Non-separable cases such as $\textrm{XOR}$ and $\textrm{Parity-of-3}$ are equally instructive.

These cases are informative precisely because they exhibit persistent failure: the trace accumulates refutation witnesses that cannot be eliminated, making the source of non-separability structurally visible rather than absorbed into a loss plateau.

The Boolean presets demonstrate the repair discipline in miniature. The EMNIST notebook applies the same schema to a larger finite witness table under identical closure conditions.

The repair loop

  1. Evaluate the current semantic state against the declared margin condition.
  2. Identify the earliest refutation witness to a margin violation.
  3. Apply a local integer repair whose magnitude is derived from the witness’s slack.
  4. Re-evaluate: check whether any finite refutation witness remains.

For state $\Sigma_t=(W_t,S_t)$, the next witness is selected by its earliest example index $i^*$; $c^*$ is the strongest competing class under the fixed tie-broken readout:

$$ \begin{array}{l} \Sigma_0 := (W_0,S_0) = (0,0),\\[0.6ex] \Sigma_{t+1} := \mathsf{Repair}_{\gamma,\Delta}(\Sigma_t),\\[0.8ex] i^* = \min\{\, i < N \mid \exists c\neq y_i:\ S(i,y_i) < S(i,c)+\gamma \,\},\\[0.4ex] c^* = \arg\max\nolimits^{\!*}_{c\neq y_{i^*}}\, S(i^*,c). \end{array} $$

The trace is not merely a visualisation aid. It is the finite, replayable history from which the terminal state and its decidable predicates can be independently recomputed and verified.

Formal setup

Fix a deterministic preprocessing pipeline, a feature map $\Phi$, a finite witness table, and a stable tie-broken $\arg\max^{*}$ under a semantics lock $\mathcal{L}$. The learner state is a sparse integer weight table $W$ together with a cached score matrix $S$ maintained by postings propagation. Training selects the earliest refutation witness to a declared margin violation, applies a local integer repair, and terminates when every margin obligation in the finite table is satisfied.

In the current implementation, the evaluator can be understood as a lambda-definable perceptron operating under the declared semantics lock.

Design aims:

  • Object. Learning transforms a finite interpretation whose semantics are explicitly declared and held invariant.
  • Method. Each update eliminates a concrete refutation witness while preserving decidable state invariants.
  • Gate. Accept the repaired evaluator only when finite validation under the lock $\mathcal{L}$ returns no remaining refutation witness—that is, when the predicate FULL.ok holds.

The lock makes explicit what is conventionally left implicit: preprocessing, feature extraction, tie-breaking rule, scan order, and evaluation criteria. Removing these hidden degrees of freedom is a precondition for any rigorous comparison of training methods.

Finite means the obligation set is bounded and enumerated. Replayable means the trace can be re-executed from any stored state. Checkable means the state predicates can be recomputed independently, without trusting the training procedure.

Training manipulates a finitary, explicitly represented state

$$ \Sigma \;\equiv\; \{\;W, S, ...\;\}, $$

where $W$ is a sparse integer weight table and $S$ is a cached score matrix (maintained via postings). This is the current prototype’s state representation, not a limit of the general idea. After the repair loop is understood, each snapshot can also be checked by decidable predicates with diagnostic witnesses on failure:

$$ \begin{array}{l|l} \mathsf{WF}(\Sigma) & \text{Well-formedness of the finite representation}\\[0.4ex] \mathsf{CACHE.ok}(\Sigma) & \text{Cache correctness}\\[0.4ex] \mathsf{FULL.ok}(\Sigma) & \text{Zero-violation / margin closure by } \arg\max^{*}\\ \end{array} $$

These are state predicates, not performance metrics. They assert, respectively, that the finite representation is well-formed, that the cached scores remain consistent with the definitional computation, and that all declared margin obligations on the finite table have been satisfied.

WF asserts that the finite state has the correct shape: weight dimensions, example count, feature count, class count, and cache entries are mutually consistent.

CACHE.ok asserts that the fast cached scores agree with the definitional recomputation from $W$ and the feature sets. This separates the semantic content of the learned object from implementation details of the cache.

FULL.ok asserts that, under the locked readout and margin rule, the finite witness table contains no remaining refutation witness. When this predicate holds, the training obligation has been discharged.

When a predicate fails, the system returns a concrete diagnostic witness: an explicit index and class pair exhibiting a cache discrepancy, or, for FULL.ok, the earliest remaining refutation witness. When all three predicates hold simultaneously, they constitute a finite certificate about the learned artifact under $\mathcal{L}$. This is not a global generalisation claim; it is a decidable assertion about the declared finite table.

Operationally, the method resembles error-driven procedures such as the perceptron, but its conceptual interpretation differs. No gradient is computed; no pre-given topological landscape is assumed. Instead, the learner receives a named refutation witness to a declared semantic obligation and performs a repair whose effect is propagated explicitly through the postings structure. Learning is thus constituted as a sequence of witness eliminations over a finitely named object. Implementation substitution of the learned evaluator is admissible only after the resulting state has satisfied its declared predicates.

Let $\textsf{E}_{\mathsf{ref}}$ be a reference evaluator induced by the lock $\mathcal{L}$ (e.g. recomputation from $\Phi$ and a declared scoring rule), and let $\textsf{E}_{\mathsf{learn}}$ be the learned evaluator encoded by a terminal snapshot $\Sigma_T$. Here, “replace” means implementation substitution: using $\textsf{E}_{\mathsf{learn}}$ where $\textsf{E}_{\mathsf{ref}}$ would otherwise be evaluated. Before that substitution, a bounded comparison must establish semantic agreement on the locked finite table, such as

$$ \forall i < \mathrm{N} : \textsf{E}_{\mathsf{learn}}(x_i) = \textsf{E}_{\mathsf{ref}}(x_i) $$

This obligation establishes equivalence only on the declared finite table; it does not identify the two implementations or justify behavior outside the lock. Acceptance of the learned state is a separate question governed by the declared closure and state predicates. When the substitution check fails, it returns the concrete failing index $i$.

Limits

The present prototype is intentionally simple in its extensional scope. Its function is to isolate the training discipline in a setting where every step is fully explicit: deterministic refutation, finite integer repair, replayable trace, and decidable closure.

The framework does not dissolve the problem of intensional opacity. Once learning amounts to selecting a program in a sufficiently expressive language, the intensional structure of that program remains underdetermined by its extensional behaviour. Where the underlying learning problem involves underdetermination, intractable minimisation, or undecidable properties, increases in model complexity do not remove that core difficulty.

Research direction

Refutational Learning is a natural fit for regimes in which a semantic object is incrementally refined in response to new refutation witnesses, and in which the transparency and verifiability of each repair step are requirements rather than incidentals. The open question is which components of this training discipline, decidable closure, explicit witness retention, replayable trace, remain tractable as the scope of the witness table and the expressive power of the model class are extended.
