Summary

CARRYLESS PAIRING

We provide a pairing function for natural numbers in the Fibonacci basis. It separates the Zeckendorf supports of $x$ and $y$ into even and offset odd bands, so encoding and decoding use addition, bounded search, and primitive recursion without carrying.

The construction below uses only addition and bounded recursion. It exposes the core mechanism first, then turns that mechanism into a pairing function.

(i) First, chosen Fibonacci terms are aggregated into a single numerical value.

(ii) The chosen indices are then discarded, and only the resulting sum is returned to a Greedy Algorithm.

(iii) Finally, the original support is recovered from the sum alone within a bounded number of steps.

This round trip, from arbitrary input to canonical additive form and back, uses Binet’s Formula, Hurwitz’s Theorem, and Zeckendorf’s Theorem to obtain a stable, reversible, and fully bounded finitary encoding-decoding scheme.

(iv) The same structure realizes a constructive, hence injective, arithmetical pairing function. Set $Z(0):=\varnothing$ and let $x, y \in \mathbb{N}$:

For $x,y\in\mathbb{N}$ we define the even-band and odd-band supports,

$$\begin{align} E(x) &:= \{\, 2k : k \in Z(x) \,\},\\ r(x) &:= \min\{\, m \ge 2 : F_m > x \,\},\\ B(x) &:= 2r(x),\\ O(x,y) &:= \{\, B(x) + (2j - 1) : j \in Z(y) \,\}. \end{align}$$

Carryless Pairing (recommended when $0$ is reserved, e.g. for $\varnothing$) is the function $\pi_{\mathrm{CL}}:\mathbb{N}^2\to\mathbb{N}$ given by:

$$\pi_{\mathrm{CL}}(x,y) \;:=\; F_2 \;+\; \sum_{k\in Z(x)} F_{2k} \;+\; \sum_{j\in Z(y)} F_{\,B(x)+(2j-1)}.$$

Equivalently,

$$\pi_{\mathrm{CL}}(x,y)=F_2+\sum_{i\in E(x)\cup O(x,y)} F_i,$$

The offset term $F_2$ ensures

$$\pi_{\mathrm{CL}}(x,y)\ge 1, \pi_{\mathrm{CL}}(0,0)\neq 0.$$

Remark. The pairing $\pi_{\mathrm{CL}}^\tau$ is finitary; correctness rests on the uniqueness and stability of Zeckendorf decompositions under exact order (cf. phi-adic logic). The apparent division in the inverse is only a $\mathcal{O}(\log n)$ search. In a numerical model that is not an exact copy of $\{\,\mathbb{N},\, +,\, \le\,\}$ (with $\Delta I$), these invariants may fail; IEEE-754 floating point past $2^{53}$ is a simple example. The point is not implementation detail but a persistent bias in contemporary computation: exact arithmetical structure is often simulated inside approximate hardware.
