Overview

CSS2Letter is a source-first DIN A4 text composer. There is no separate authoring interface: the letter is composed by editing composer/index.html directly and refreshing it in the browser. The sender, recipient, metadata, and paragraph-coded letter text all remain visible in the HTML.

This is a small example of moldable development: the writer works on the inspectable artifact that shapes the document instead of adapting the document to a fixed editor. Content, typography, spacing, and page geometry remain exposed as source. Paragraph codes control pagination, so the division into sheets is recomputed after the text changes; the browser then produces the print-ready PDF.

Illustration: DIN A paper preserves the ratio 1:√2 under halving. An A4 sheet therefore contains A5, A6, and the subsequent formats in the same proportion.

Project Files

The project consists of a website page and a self-contained composer:

  • index.html presents this overview, navigation, and project references.
  • css2letter.css contains the recursive DIN proportion illustration.
  • composer/index.html is the editable composition source. It contains the sender, recipient, metadata, letter paragraphs, DIN 5008 template, and print command.
  • composer/styles/din5008.css controls A4 geometry, typography, letter fields, screen preview, and PDF print output.
  • composer/scripts/pagination.js measures the available page area and moves or divides content across additional sheets.
  • composer/scripts/letter-engine.js waits for fonts and mathematics, then starts pagination for the HTML letter content.
  • composer/assets/logo.png and signature.png provide the visible letterhead artwork.
  • composer/assets/reference/din.png and mr.png are visual references for template development.
